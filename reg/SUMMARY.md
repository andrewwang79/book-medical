# 医疗器械注册
* 三点：真实，安全，有效
* QMS
* DHF，DMR，DHR三者共同确保了医疗设备从设计到交付使用的全过程都符合质量和法规要求。

## 术语
| 术语 | 全称 | 说明 |
| :-: | - | - |
| DHF | Design History File <br/> 设计历史记录文档 | **医疗器械设计开发活动的所有纪录。证明设计过程遵循规定的设计控制过程。**<br>包含策划、客户需求、设计输入输出、验证、确认，设计评审、设计转换，设计变更的所有记录。 |
| DMR | Device Master Record <br/> 器械主记录 | **器械制造的蓝图，制造和测试医疗器械所需的一切程序和规范**，确保器械每次都能以一致的方式被正确的制造。<br>包含器械规范、生产加工规范、品质保证程序和规范、包装和标记规范、安装维护服务的程序和方法。 |
| DHR | Device History Record <br/> 器械(生产)历史记录 | **每台器械成品制造过程的记录，证明了器械是按照DMR规定制造的。**<br>包含制造日期、数量、放行销售数量、验收记录、标签标记、唯一器械标识符。 |
| 型检 | 型式检验 | 医疗器械注册检验，即产品全性能试验，型检合格的产品方可进行临床试验或申请注册。 |
| 临床评价 |  | 是指采用科学合理的方法对临床数据进行分析、评价，以确认医疗器械在其适用范国内的**安全性、有效性**的活动。 |
| 临床试验 |  | 临床评价的方式之一，评价医疗器械的安全性、临床性能和有效性，在一例或多例受试者中开展的系统性的试验或研究。FIM（Field Investigator Meeting，研究者会议） |
| 体考 | 体系核查 | 监管部门对于注册许可申请人和生产许可申请人的生产质量体系核查，以确认医疗器械设计开发和生产满足《医疗器械生产质量规范》。 |
| PMS | Post-Market Surveillance 上市后监督 | 包含上市后产品质量监督、合规监督、使用反馈数据等分析，以评估是否需要执行纠正和预防措施。 |
| 飞检 | 飞行检查 | 监管部门对医疗器械企业开展的不预先告知的监督检查。 |
| 国抽 | 质量抽查检验 | 对医疗器械产品质量的抽查检验工作。 |

## 注册单元和型号
1. 发证规则：1 个注册单元对应 1 张医疗器械注册证
1. 同一注册单元判定标准
    * 产品核心原理、整机结构、关键性能指标、临床适用范围完全统一；
    * 仅差异为：非附加性功能、选配硬件、软件授权模块、非关键零部件等不影响安全有效项。
1. 型号与注册证关系：一张注册证支持多型号（1:N）
    * 示例：主系列为同一注册单元，区分不同子型号，依靠选配硬件/软件授权区分配置。

### 1个注册单元的多型号方案
1. 方案：核心硬件统一标准化，区分型号依靠可选配件 + 软件授权模块。生产过程时通过工具软件配置型号。
1. 各环节注意事项
    1. 体系文档：产品规划文件完整列明全部子型号；需求文件覆盖全系列所有功能（各软件授权模块单独作为需求条目）；测试工作覆盖全部授权模块开关两种场景。
    1. 生产：每一种型号单独完成成品验，留存各型号专属检验记录。

| 环节 | 关键动作 | 合规目的 |
| :-: | - | - |
| 产品规划文档 | 清晰定义全部子型号、各型号差异点、软件授权模块对应关系 | 全系产品属于同一注册单元，不存在超注册范围设计 |
| 需求文档 | 完整收录全系列所有功能，各软件授权模块单独拆分需求项 | 全部功能可追溯 |
| 测试文档 | 针对每个授权模块单独设计测试用例，覆盖模块开启、关闭两种工况 | 各型号配置对应的功能均完成验证，授权逻辑无漏洞 |
| 生产制造 | 留存各型号出厂配置清单、对应成品检验原始记录 | 审计/核查时可证明每个型号与注册申报一致 |

## 临床
### 免临床试验的三种途径
1. 产品在临床豁免目录
1. 临床评价后免临床试验
1. 同品种医疗器械比对

### 同品种医疗器械比对
* 同类比对需要功能的适用范围和构成相同，产品的不同功能可以找不同的竞品做同类比对
* 公司上一代产品是传统算法，新一代采用了AI算法。如果上一代做过临床实验，可以做比对
* 比对来源
  1. 公司内部产品：如上一代
  1. 外部公司授权[死路]：没公司会同意授权
  1. 科研合作试用竞品[死路]：竞品算法结果拿不到，没法比对

### 新功能、算法和用途的处理
1. 辅助决策：原则上均需开展临床实验
1. 非辅助决策：原则上均需开展临床评价

* [数据生命周期](https://ai.wangyaqi.cn/#/kb/data)

## 外采设备
* 外采设备如能量设备，同系列产品的分析方法：
    1. 选择风险最高、结构最复杂、功能最全面的型号，做典型性分析
    1. 对非典型型号的特殊属性（如接口差异、参数微调），仅做针对性的适配性验证，而非全流程评估
* 外采设备注册写法

| 类型 | 写法 |
| - | - |
| 通用设备(直接使用) | 写硬件接口及其标准，不限数量 |
| 专用设备(通过API管理) | 写供应商的设备注册证，不写具体型号，尽量全系列 |
